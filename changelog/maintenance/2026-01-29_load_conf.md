## Config loader migration (2026-01-29)

### Goal
Unify config loading so scripts are:
- template-visible (tracked examples in `conf/*.env.example`)
- runnable per-machine (real configs in `ops/*.env`, gitignored)
- sync-safe (no private paths committed)

### New convention
- Templates: `conf/<module>.env.example`
- Runtime configs (local only): `ops/<module>.env`
- Loader: `scripts/_lib/load_conf.sh` with `load_module_conf <module> <required keys...>`

### Migration checklist
1. Replace legacy `ENV_FILE/CONF_PATH` manual path logic with `load_module_conf`.
2. Move machine-specific values into `ops/<module>.env`.
3. Add/refresh `conf/<module>.env.example` (sanitized).
4. Tag outputs by scope: `[LOCAL]`, `[NETWORK]`, `[REMOTE]`.
5. Run `bin/scan_conf_migration.sh` until no LEGACY hits remain.

### Notes
- `open_last_snapshot.sh` opens local log immediately; remote open happens only if local snapshot dirs are missing.
