Personal CLI toolbox (single source of truth: `scripts/`, launchers in `bin/`).
toolbox_super_compatible 是主入口
其它旧入口已废弃（legacy）

改逻辑只动 scripts；改入口只动 bin；产物只进 _out

写进 README/贡献指南都行（一句话就够）：
证据图/附件一律放 changelog/_assets/（或事件目录内）
严禁 file:// 绝对路径
_out/ 只放运行产物，不放证据
## Layout

- `scripts/` — source of truth (all real logic lives here)
- `bin/` — launchers only (tiny exec wrappers -> scripts)
- `_lib/` — shared shell helpers
- `conf/` — config templates / defaults (avoid committing secrets)
- `ops/` — incubation / one-off utilities (promote stable ones into `scripts/`)
- `_out/` — runtime outputs (ignored by git)
- `changelog/` — update history (release notes)

## 为什么这样改对你更“顺手”
1) `PATH` 从硬编码绝对路径改成 `$HOME`（你未来换机器/目录不会炸）。  
2) 明确 `bin=launchers only` + `scripts=source of truth`，把你刚建立的工程约束写进“宪法”。  
3) 把更新入口写出来（changelog/tag），让 repo 变成“可查询的产品”。
